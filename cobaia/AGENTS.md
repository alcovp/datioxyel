# ТЗ: MVP Ray Tracer (статичная сцена)

## Цель
Сделать простой офлайн-рендерер, который по запуску скрипта строит **один** кадр, сохраняет его на диск и завершает работу.

## Формат запуска
- Запускается один скрипт (CLI).
- Скрипт рендерит ровно одно изображение.
- После сохранения файла скрипт завершает процесс.

## Ограничения и допущения
- Сцена полностью статична.
- Сцена, материалы, камера и свет задаются в коде (hardcoded), без внешнего формата сцен.
- Параллелизм: минимальный, без специальных оптимизаций.
- Ограничение трассировки для отражений/рефракций: глубина прохода `2`.

## Требуемые возможности рендера
- 3D-сцена.
- Прозрачные объекты с поддержкой:
  - рефракции,
  - отражения.
- Источники света присутствуют и задаются в коде.

## Сцена (первая версия)
- Главный объект: 3D-фигура **губка Менгера**.
- Материал губки: стекло, немного мутное.
- Положение камеры: под углом к объекту.
- Композиция: губка занимает существенную часть кадра.
- Основание: плоскость-пол, цвет почти белый (чуть темнее белого).
- Свет: один источник, имитация солнца.

## Выбор языка и стека
- Принято: **Node.js (TypeScript) + Rust**.

### Разделение ответственности
- `Node.js/TypeScript`: CLI, конфигурация запуска, описание сцены, управление пайплайном.
- `Rust`: вычислительное ядро трассировки лучей (reflection/refraction), дальнейшее масштабирование по параллелизму.

### Почему этот выбор
- Сохраняем удобство разработки и сценарного запуска через Node.js.
- Получаем производительность и безопасный параллелизм в Rust для роста сложности сцены и рендера.
- Архитектура хорошо масштабируется: Node как orchestration-слой, Rust как render engine.

## Критерий готовности MVP
После запуска скрипта на диске появляется один файл изображения с рендером описанной сцены, и процесс завершается без дополнительных действий пользователя.
