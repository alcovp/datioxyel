# AGENTS

Этот репозиторий нужен только для проверки `codex exec parallel` возможностей.
Планируются жесткие тесты с десятками (до 30) параллельных запусков Codex.
Все задачи выполняются над подопытным проектом в папке `cobaia`, созданным специально для этого.
Здесь будет несколько независимых проверок: тесты разных гипотез о bottleneck-точках, из-за которых параллельные запуски могут фейлиться.
Эти проверки можно запускать как бенчмарки по необходимости.
Первый бенчмарк-сценарий: массовый запуск `codex exec` для code review проекта `cobaia` с сохранением каждого отчёта в отдельный `.md` в `out/parallel_review_baseline`.

## Подтверждённые результаты (2026-02-22)

- Run `20260222_165647` (`parallel_review_baseline`, 30 воркеров, без stagger): `success=15`, `failed=15`; в failed-логах общий финал — `429 Too Many Requests`.
- Run `20260222_172816` (`parallel_review_baseline`, 30 воркеров, со stagger запуска 1-10 сек): `success=30`, `failed=0`.
- Вывод: ключевой bottleneck при `30` — не общий объём работы, а стартовый burst запросов к API.
- Практика для стресс-прогонов на `30`: использовать запуск со stagger (`scripts/run_parallel_review_baseline_stagger.sh`) или эквивалентный jitter старта в новых сценариях.
